# 웹 변천사



## 1. Servlet
 - 클라이언트가 어떠한 요청을 하면 그에 대한 결과를 다시 
 전송
 - Servlet 소스에서 HTML을 처리함
 - 비지니스 로직과 프리젠테이션 로직이 분리되지 않음

## 2. JSP
- JSP 모델-1
- Java Server Page
- Servlet 소스에서 처리하던 HTML을 JSP 스트립팅 기술로 대체



## 3. JSP스크립팅 기술의 한계
- 여전히 비지니스 로직과 프리젠테이션 로직 분리가 어려움


## 4. MCV 패턴
- JSP 모델-2 방식
- **Model** : 자바클래스(DAO,DTO)
- **View** : JSP,JSTL
- **Controller** : Servelt


## 5. Framework
- 스트러츠
- 스프링
- 하이버네이트
- 아이바티스
